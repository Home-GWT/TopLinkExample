> Рассказать об полиморфизме, инкапсуляции и наследованию в Java ?
  http://java-se-learning.blogspot.com/2013/07/blog-post_4.html
  Парадигмы объектно ориентированного программирования Java основаны на трёх основополагающих концепциях, называемых абстракция, инкапсуляция, наследование и полиморфизм.
  1. Абстракция - выделение важных особенностей, характеристик объекта, главным образом отличающего его от других объектов. Выделение главных особенностей, не вдаваясь в в подробности, не вдаваясь в детали.
  2. Полиморфизм - возможность одного интерфейса иметь различные реализация. Одна спецификация - много реализация. Для Java полиморфизм - это исключительно позднее связывание (runtime binding).
  3. Инкапсуляция - сокрытие реализации. Это значит что для объекта, который хочет сотрудничать(работать) с классом видны только поля, доступные в соответствии модификаторов доступа, и спецификации методов , доступные в соответствии модификаторов доступа. Реализация методов недоступна.
  4. Наследование - возможность создавать обобьет на основании другого объекта. Это означает что новый обобьет создается не "с нуля", а с шаблона родителя - набор свойств  и поведение (набор методов) будет унаследован от родителя.

состоит из 5-ти компонентов:
1. auth-сервера (протокол OAuth2)
2. offer-сервера (RESTFull, Spring-4, Java-7/8)
3. message-сервера (RabbitMQ при поддержке в Spring-а)
4. веб-клиента (на Angular-2)
   это "Adver" или "Доска объявлений" (тестовый) http://gup.com.ua:4000
5. мобильного-клиента (Qt версия-5)

Базы данных: PostgreSQL(8), MongoDB(3.2)
ElasticSearch (with MongoDB)

Проекту уже 2-года.
Год назад когда я пришел, этот проект имел монолитную структуру, в котором были только базовые модули. А его фронтальная часть была реализована на JSP.
Первым делом я поделил проект на модульную структуру и создал микросервисную архитектуру.
Установил и настроил билд-платформу на базе Jenkins.
Перенастроил тестовые + боевые мидл-вары и базы данных. 
Настроил балансировку для мидл-варов, использовал NGinx.
Настроил репликацию для базы данных MongoDB.
Добавил таски по бэкапу баз данных.

- Внедрил RESTFull подход для веб-методов.
- Прикрутил Swagger для автоматического документирования веб-методов.
- Подключил регистрацию/авторизацию через социальные сети.
- Внедрил поиск по геолокации (с долготой, широтой).
- Внедрил полнотекстовый поиск через ElasticSearch.
- Добавил систему пересчета валют.
- Код покрывался тестами (JUnit, интеграционными тестами)
  используя Spring RestTemplate, Fongo

Реализовал серверную часть message-сервера на базе RabbitMQ (при поддержке Spring-а) - сама задача разбита на подзадачи:
- разработка структуры таблиц в реляционной базе данных; написание процедур для PostgreSQL;
- и механизм доставки клиентских сообщений через RabbitMQ.

Еще участвовал в разработке безопасной системы оплаты реализованой на "биткойнах" - это такие себе "умные контракты" в "криптовалюте".
Сеть "биткойнов", внутри которой хранятся цепочки блоков - это "блокчейн".
Передача средств между биткойн-кошельками - это "транзакция" - информация о которой включается в цепочку блоков (блокчейн)
(Майнинг - это держатели серверов для сети "биткойнов")
